1 # BNO055 IMU driver
2 
3 This driver is meant to control the BNO055 IMU device, in a ChibiOS project.
4 It provides functions for a simple use of this component, but can be extended
5 in the future to provide other functions, according to the needs.
6 
7 ## Installation manual:
8  In order to include this driver in your project, perform the following steps:
9  - Set the USE_BNO055 symbol to TRUE in the drivers.mk file
10  - Include the drivers.mk file in the main Makefile of your project (use the
11  other includes as model to do this)
12  - Add $(DRVSRC) in CSRC and $(DRVINC) in INCDIR
13  And that's all. Easy, no?
14  Simply include imudriver.h in your source code files to use the API.
15 
16 ## User manual:
17  - Call the **initIMU()** function first, as it configures all what is required
18  for a proper use of the device.
19  This function set the device in IMU fusion mode, which allow the user to
20  access to most of the useful data: euler angle, quaternions,...
21  - Call the **setHeading()** function to fix the initial heading of the device.
22  - Then, call the **getHeading()**, **getRoll()** & **getPitch()** function.
23 
24 ## Documentation
25  To generate a complete documentation of the API, you can use Doxygen.
26  Simply run ```doxygen Doxyfile``` in the root directory of the repository and
27  the documentation will be generated in the **/doc** directory.
28  Open **/doc/html/index.html** with your favorite web browser to read the
29  doc.

31 ## For the robotic cup:
32  When initialising the robot, once we know which color (and thus side) we are,
33  set initial heading to :
34  - alpha if color1
35  - (alpha + 180) % 360 (in degrees) if color2
36 
37 This way, angles will be the same in both cases.
