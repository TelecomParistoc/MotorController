27 /*
28  * @brief Initialise the motor driver.
29  *
30  * @details This function is in charge of enabling all the low-level components
31  * used to control the motors. It configures the timers used to
32  * generate the PWM and starts them. It also sets the direction pins
33  * so that the motors turn forward as defined by the parameters.
34  *
35  * @param[in] motor_left_forward_sense The rotation direction which corresponds
36  * to a forward movement of the left motor.
37  * @param[in] motor_right_forward_sense The rotation direction which corresponds
38  * to a forward movement of the right motor.
39  *
40  * @note To provide a high-level interface to control the motors, this driver
41  * uses the values FORWARD and BACKWARD to describe the rotation direction
42  * of the motors. However, these directions depend on how the motors are
43  * placed in the robot and how they are wired on the H-bridge. In order to
44  * foster reusability of this driver, the mapping between pins state and
45  * rotation direction is settable.
46  * The 2 parameters of this init function are for this purpose.
47  * DIRECTION_1 corresponds to PIN_A at '1' (+3V3) and PIN_B at '0' (GND).
48  * DIRECTION_2 corresponds to PIN_A at '0' (GND) and PIN_B at '1' (+3V3).
49  *
50  * @warning This function must be called first to start the motor driver.
51  */
